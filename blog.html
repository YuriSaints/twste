❌ Maus hábitos ao escrever HTML e CSS

  • Estilos misturados no HTML (ex: `<p style="color: green;">`)
  • Uso excessivo de `<div>` sem necessidade
  • Uso de `!important` no CSS sem necessidade
  • Falta de semântica (poderia usar `<section>` ou `<article>` no lugar de `<div>`)
  • Código desorganizado e pouco reutilizável

Agora vamos falar dos bons hábitos ao escrever código. Isso pode variar de Dev para Dev, mas sempre é importante lembrar que boas práticas tornam o código mais limpo e profissional.

✅ Boas práticas ao escrever HTML e CSS

  • Estilos foram movidos para um arquivo CSS externo
  • Uso de classes reutilizáveis no CSS
  • Melhor semântica no HTML (uso de `<section>` no lugar de `<div>`)
  • Melhor organização, facilitando manutenção
